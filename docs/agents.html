What the ZK Layer Does (and Doesn't) Hide

Hidden (ZK-proven)
• Owner identity (who delegated the agent)
• Spending limits and budget ranges
• Cumulative spend caps and history
• Policy parameters and constraints
• Delegation chain details
Public (on-chain)
• Agent wallet address
• Transaction amounts and destinations
• Deposits, swaps, LP positions
• Token balances
• All tx history on BaseScan

The ZK layer proves the agent is authorized and within budget without revealing the policy behind it. Transactions themselves remain public on Base — true tx privacy would require a shielded chain (e.g., Aztec). Selective disclosure lets auditors verify compliance without seeing the full policy.
